# Zgłaszanie podatności / bezpieczeństwo

Jeżeli znajdziesz problem związany z bezpieczeństwem (podatność, wyciek danych, itp.), proszę zgłosić go poprzez mechanizmy dostępne na GitHub:

- użyj sekcji "Security" → "Report a vulnerability" (GitHub Security Advisory),
- lub otwórz issue oznaczone jako prywatne, jeśli potrzebujesz dyskrecji.

W zgłoszeniu podaj:

- krótkie podsumowanie i wpływ,
- kroki niezbędne do reprodukcji,
- środowisko/testy (jeśli dotyczy),
- dowolne dowody koncepcji (PoC).

Nie umieszczaj w publicznym zgłoszeniu danych wrażliwych (kluczy, haseł). Jeśli potrzebujesz przesłać takie informacje, skontaktuj się najpierw przez GitHub Security Advisory lub inny bezpieczny kanał.

Odpowiedź właściciela repo może zająć do kilku dni roboczych. Jeżeli zgłoszenie dotyczy krytycznej luki, oznacz to wyraźnie.
